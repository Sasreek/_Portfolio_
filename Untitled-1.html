Design Case Study: Wellness Tracker App Concept

This case study demonstrates my ability to apply user-centered design principles to solve a real-world problem, focusing on the journey from problem discovery to final visual design.

Phase 1: Discovery & Problem Definition

**Problem:** Users struggle with scattered health data across multiple apps (sleep, fitness, mood). The goal was to consolidate tracking into a single, cohesive, and motivating interface.

  • Conducted user interviews to identify key pain points (data entry friction, lack of consolidated feedback).
  • Defined the core user group: busy professionals seeking simple, automated tracking.
  • **Solution Goal:** Design a minimal, motivational dashboard providing actionable insights at a glance.

Phase 2: Wireframing & Prototyping (Figma)

Used Figma to create low-fidelity wireframes, focusing purely on **Information Architecture** and flow before introducing visual style.

  • Designed a primary navigation focused on three areas: Dashboard, Input, and History.
  • Created an interactive prototype to test the user flow for logging sleep and mood, ensuring minimal steps.
  • Iterated based on usability testing feedback, optimizing icon placement and reducing text clutter.

Phase 3: Visual Design & Handoff

Applied a calming, accessible color palette (dark mode primary) and established a clear design system.

  • Utilized bold, accessible typography (Inter font) and high-contrast color pairings for charts.
  • Designed custom data visualization components (progress rings, historical line charts) to maximize information density without sacrificing visual clarity.
  • Prepared a final style guide and asset package for developer handoff, including specific component states and responsive breakpoints.
